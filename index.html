Fundamentals

Importance of Cyber Security

CIA Triad — Confidentiality, Integrity, Availability. The invisible shield of the digital age.

CIA TriadInfoSec
Open note →

Cybersecurity is essential because our lives are increasingly lived through data. It safeguards systems that drive economies and institutions — keeping hospitals functional, banks operational, and communication networks reliable.

The CIA Triad defines the three fundamental principles:

  • Confidentiality — Only authorized eyes see sensitive data (patient records, payment data, cloud files)
  • Integrity — Data remains untampered (document platforms, database records, online reviews)
  • Availability — Systems stay online (hospital networks, cloud services, banking systems)

A violation of any pillar weakens the whole structure — unauthorized access breaches confidentiality, data corruption compromises integrity, and outages kill availability.

Fundamentals

Introduction to Web Application Security

Evolution from ARPANET to the modern web. OWASP Top 10, URL anatomy, defense checklists.

OWASPWebDefense
Open note →

Web Application Security protects websites and online platforms from unauthorized access, misuse, or disruption.

Web Evolution: ARPANET (1969, research-only) → Old Web (1990s, static pages) → Modern Web (dynamic, cloud-powered, primary attack target).

Proactive Defense Checklist:

  • Enforce MFA everywhere
  • Regular penetration testing + vuln scans
  • Patch everything, always
  • Network segmentation
  • Test your incident response plan

Case Study — BlackCat (ALPHV): Feb 2024 detection → Mar 2024 data exfiltration confirmed → Jan 2025: 190M records exposed.

The OWASP Top 10 is the universal benchmark for web app risks — regularly updated with real-world breach data.

Fundamentals

Encryption Fundamentals

Hash functions, ciphers, key exchange. Certificate Authorities, X.509, and digital trust.

CryptoPKITLS
Open note →

Encryption converts readable information into ciphertext, accessible only with the correct key. It ensures confidentiality, integrity, and trust across digital communications.

Core components:

  • Hash functions — generate unique fingerprints of data for verification
  • Ciphers — encrypt and decrypt using mathematical operations
  • Key exchange algorithms — securely share encryption keys between parties

End-to-end encryption — only communicating parties can read messages.
Transport encryption — secures data only during transmission.

Certificate Authorities (CAs), Trusted Root-CAs, and X.509 Certificates create the chain of trust that lets you verify legitimate websites and services.

Fundamentals

Cloud & Virtualization Fundamentals

Public, private, hybrid cloud. Hypervisors, VMs, CASB. The invisible foundation of the internet.

CloudVMsCASB
Open note →

Cloud — a network of servers providing storage, software, and computing over the internet. Instead of owning hardware, you rent resources, gaining flexibility while reducing infrastructure costs.

Virtualization — enables multiple OS instances on a single physical machine via a hypervisor. Each VM gets isolated CPU, memory, and storage allocations.

Deployment models:

  • Public Cloud — shared resources (AWS, Azure, Google Cloud)
  • Private Cloud — dedicated infrastructure, greater control
  • Hybrid Cloud — balanced mix of both

CASB (Cloud Access Security Broker) — sits between users and cloud services to monitor data movement, enforce policies, detect threats, and ensure compliance.

CWE-78

OS Command Injection

When a web form becomes a root shell. Semicolons, pipes, and $() hand the server to the attacker.

RCEShellInjection
Open note →

OS Command Injection allows an attacker to execute arbitrary OS commands via a vulnerable web application — giving full control over the server.

It happens when user input is directly inserted into system commands without sanitization. Characters like ; | && $() break out of the intended command context.

Example attack:
App runs: cat report.txt
Attacker inputs: report.txt; rm -rf /

Why it's devastating: works without direct server access. If the app runs as root — game over.

Prevention: never pass raw user input to system shells. Use parameterized APIs, input allowlisting, and principle of least privilege.

+

CWE-89

SQL Injection

The crown jewel of web vulns. OR 1=1 login bypass, data theft, full DB compromise.

SQLiAuth BypassDB
Open note →

SQL Injection occurs when a web application builds a database query using unvalidated user input, allowing attackers to manipulate SQL commands.

Classic login bypass: Enter ' OR '1'='1 as the username — the WHERE clause becomes always-true, granting access without a real password.

Impact:

  • Data breach — steal credentials, PII, payment info
  • Data loss — modify or delete records
  • Full server compromise via DB in extreme cases

Prevention:

  • Use prepared statements / parameterized queries
  • Whitelist allowed characters on input
  • Never run the database as root/admin
  • Hide real error messages from users

CWE-79

Cross-Site Scripting (XSS)

Reflected, Stored, DOM-based, uXSS. Session hijacking and account takeover via the browser.

XSSCSPSessions
Open note →

XSS lets attackers inject malicious scripts that execute in victims' browsers. The browser trusts the script because it appears to come from the legitimate site.

Types:

  • Reflected (Type 1) — script reflected immediately via crafted link
  • Stored (Type 2) — script saved in DB, runs for every visitor
  • DOM-Based (Type 0) — attack is entirely client-side
  • uXSS — exploits browser/plugin vulnerabilities

Consequences: session hijacking, data theft, account takeover, malware distribution, website defacement.

Prevention: output encoding (context-aware), input validation, Content Security Policy (CSP), HTTPOnly cookies.

CWE-35

Path Traversal

Walking outside the web root with ../ — encoding tricks, null bytes, and allowlist defenses.

../File AccessBypass
Open note →

Path Traversal allows attackers to access files and directories outside the intended application directory by manipulating file path input.

Common techniques:

  • ../ — basic upward traversal
  • %2e%2e%2f — URL-encoded traversal
  • %252e%252e%252f — double-encoded bypass
  • %00 — null byte injection (legacy APIs)
  • ....// — mangled path normalization tricks

Consequences: read /etc/passwd, database credentials, source code — or modify/delete critical files.

Prevention: allowlist valid filenames, use realpath() / Path.resolve() then verify result is inside allowed directory, run app with least privilege, isolate with containers/chroot.

Burp Suite Fundamentals

Proxy, Repeater, Intruder, Scanner. Sniper vs Cluster Bomb. DVWA as the perfect legal lab.

BurpProxyDVWA
Open note →

Burp Suite is the gold-standard platform for web application security testing — used by pentesters, bug bounty hunters, and red teams worldwide.

Core tools:

  • Proxy — intercept & modify HTTP/S traffic in real time
  • Repeater — manually edit and resend individual requests
  • Intruder — automate brute-force, fuzzing, and payload attacks
  • Scanner — automated vulnerability detection (Pro only)
  • Sequencer — analyze randomness of tokens & session IDs

Intruder attack types:

  • Sniper — fuzz one parameter at a time
  • Battering Ram — same payload in multiple positions
  • Pitchfork — different payloads per position
  • Cluster Bomb — all combinations (full brute force)

Quick setup: set browser proxy to 127.0.0.1:8080, install Burp CA certificate, toggle Intercept ON. Pair with DVWA for a safe, legal practice environment.

Career Opportunities in IT Security

Red vs Blue team, SOC Analyst, Pentester, DFIR. The full pentest lifecycle from recon to report.

Red TeamBlue TeamSOC
Open note →

Red Team (Offensive) — simulate real attacks, exploit, social engineering. Tools: Metasploit, Cobalt Strike.

Blue Team (Defensive) — detect, monitor, respond, harden. Tools: SIEM, EDR, firewalls.

Expanded roles:

  • Penetration Tester — simulates real attacks with proof of impact
  • SOC Analyst — 24/7 monitoring and triage
  • Threat Hunter — hunts evil before detection triggers
  • DFIR Analyst — investigates breaches, preserves evidence
  • Malware Reverse Engineer — tears apart malicious code
  • Security Researcher — discovers zero-days, publishes CVEs

Pentest lifecycle: Recon → Scanning → Gaining Access → Maintaining Access → Covering Tracks → Reporting.

Recommended certs: OSCP, PNPT, CompTIA Security+, eJPT.

Email Security Fundamentals

S/MIME, OpenPGP, SPF, DKIM, DMARC. Phishing, BEC, spoofing — your inbox is the front door.

DMARCPhishingBEC
Open note →

Four pillars of email security: Confidentiality, Integrity, Authentication, Non-repudiation.

Encryption standards:

  • S/MIME — enterprise standard, uses certificates from trusted CAs
  • OpenPGP — open-source, Web of Trust model, privacy-focused

Common threats:

  • Phishing — deceptive emails fishing for credentials
  • Spear Phishing — targeted, personalized phishing
  • Email Spoofing — forged headers to impersonate trusted senders
  • BEC (Business Email Compromise) — impersonating executives to initiate fraud

Authentication protocols:

  • SPF — verifies sender IP against domain's authorized list
  • DKIM — cryptographic signature verifies message authenticity
  • DMARC — policy for handling SPF/DKIM failures

Networking

Network Traffic Analysis

Wireshark, tcpdump, Tshark. Capture vs display filters. Every packet tells a story.

WiresharkNTAPackets
Open note →

Network Traffic Analysis (NTA) — capturing, inspecting, and analyzing network data to characterize ports/protocols, establish baselines, and respond to threats.

The 4-step NTA process:

  • Ingest Traffic — capture with filters to reduce volume
  • Reduce Noise — filter benign/irrelevant traffic
  • Analyze & Explore — carve out data relevant to the issue
  • Detect & Alert — identify malicious patterns, use IDS

Essential tools:

  • Wireshark — graphical deep packet inspection, layer-by-layer decoding
  • tcpdump / Tshark — command-line packet analyzers
  • Network Taps / SPAN ports — copy traffic for external analysis

Wireshark filter types:
Capture filters (e.g. tcp port 80) — applied before capture.
Display filters (e.g. ip.addr == x.x.x.x) — applied after capture.
