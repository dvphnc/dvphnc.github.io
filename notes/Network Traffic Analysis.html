Network Traffic Analysis

Capture. Filter. Analyze. Dominate the wire.

⌛ reading... dvphnc

Network Traffic Analysis (NTA) is the act of capturing, inspecting, and analyzing network data flow to characterize ports and protocols, establish a performance baseline, and monitor/respond to security threats.

The NTA Process in 4 Steps

  1. Ingest Traffic: Capture network data, utilizing capture filters as needed.
  2. Reduce Noise by Filtering: Filter out unnecessary or benign traffic for cleaner analysis.
  3. Analyze and Explore: Carve out data relevant to the current issue.
  4. Detect and Alert: Look for errors and use analysis to determine if traffic is malicious, leveraging tools like Intrusion Detection Systems (IDS).

NTA Analysis Approaches

  • Descriptive: Describes a data set based on existing characteristics.
  • Diagnostic: Clarifies the causes and effects of network conditions.
  • Predictive: Creates models to forecast future probabilities (e.g., future traffic load).
  • Prescriptive: Aims to narrow down actions to take to prevent or eliminate a future problem.

Essential NTA Tools

  • Wireshark: A free and open-source graphical network traffic analyzer. Used for deep packet inspection, capturing and decoding frames to inspect protocols layer-by-layer (based on the OSI Model).
  • tcpdump / Tshark: Command-line network packet analyzers.
  • Network Taps / Span Ports: Methods (hardware or switch features) used to copy network traffic for external analysis.

Wireshark Filters

  • Capture Filters (e.g., tcp port 80): Used to filter packets before they are captured, reducing the data volume.
  • Display Filters (e.g., ip.addr == x.x.x.x): Used to filter or refine the view of packets after they have already been captured.
