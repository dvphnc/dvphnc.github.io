When a web form becomes a root shell

Command Injection happens when an attacker tricks a system into executing unintended commands — convincing a computer to follow whispers it was never meant to hear.

When user inputs aren’t properly sanitized, they can escape their intended context and reach the command shell. Suddenly, what was supposed to print a name might delete files or steal data. That’s the quiet power of injection.

Definition & Goal

OS Command Injection is a type of attack where the attacker’s goal is to execute arbitrary commands on the victim’s operating system via a vulnerable application.

  • Executed through a vulnerable web app
  • Gives attacker full control over the server
  • Can run any command the app process has access to

Why It’s So Dangerous

  • Runs arbitrary OS commands
  • Works even without direct server access
  • If the app runs as root → Game over

How It Works

User input is directly inserted into system commands without sanitization.
Characters like ;, |, &&, $() break out of the intended command. 

# Vulnerable code
user_input = "report.txt"
system("cat " + user_input)

# Attacker enters:
report.txt; rm -rf / --no-preserve-root
